## English

### How to Contribute

This project follows a structured change management process aligned with PMBOK® 8th Edition principles.

**For documentation changes:**
1. Fork the repository
2. Create a branch (`feature/your-change`)
3. Make changes in ALL three languages (EN, PT-BR, ES) or flag missing translations
4. Submit a Pull Request with a clear description

**For governance/scope changes:**
1. Open a GitHub Issue using the **Change Request** template
2. The Project Manager will assess impact and route for approval
3. Approved CRs are tracked in `CHANGELOG.md`

### Branch Naming Convention

- `feature/description` — New features or content
- `fix/description` — Bug fixes or corrections
- `docs/language/description` — Documentation in specific language
- `cr/CR-XXX` — Change request implementation

## Português

### Como Contribuir

Este projeto segue um processo estruturado de gestão de mudanças alinhado aos princípios do PMBOK® 8ª Edição.

**Para mudanças em documentação:**
1. Faça fork do repositório
2. Crie uma branch (`feature/sua-mudança`)
3. Faça alterações nos TRÊS idiomas (EN, PT-BR, ES) ou sinalize traduções pendentes
4. Submeta um Pull Request com descrição clara

**Para mudanças de governança/escopo:**
1. Abra uma Issue no GitHub usando o template de **Change Request**
2. O Gerente de Projeto avaliará o impacto e encaminhará para aprovação
3. CRs aprovadas são rastreadas no `CHANGELOG.md`

## Español

Este proyecto sigue un proceso estructurado de gestión de cambios alineado con los principios del PMBOK® 8ª Edición.

**Para cambios en documentación:**
1. Haga fork del repositorio
2. Cree una rama (`feature/su-cambio`)

2. El Director de Proyecto evaluará el impacto y lo dirigirá para aprobación
